الرئيسية تجربة النوم التحليل القاموس الرمزي الهوية التقنية التفاعل الهادئ

البنية التقنية لتطبيق SleepShare

الهدف هو بناء تطبيق هادئ تقنيًا أيضًا: بنية خفيفة، حماية قوية للخصوصية، ومرونة تسمح بتوسّع الشبكة دون أن تتحول إلى منصة جمع بيانات.

1. تقنيات التنفيذ (Tech Stack)

التطبيق المحمول

  • Framework: Flutter (لغة Dart) لنسخة موحدة على iOS و Android.
  • تصميم يعتمد على True OLED Black لتقليل استهلاك البطارية.
  • Animations بسيطة (Fade / Scale) لتجنب إجهاد العين قبل النوم.

الخادم (Backend)

  • Firebase Auth مع دعم تسجيل مجهول (Anonymous Login).
  • Firestore لتخزين الحالات والرسائل المجهولة.
  • Cloud Functions للفلاتر الذكية وتوزيع رسائل الغد بعد الاستيقاظ.

2. الكيانات الجوهرية (Core Entities)

User – المستخدم

هوية مؤقتة تتجدد يوميًا (Ephemeral ID) لتقليل إمكانية التتبع. يتم تخزين: Mood_Symbol, Chosen_Room, TimeZone, Geo_Cell واسعة.

Session – الجلسة

الفترة بين «أنا ذاهب للنوم الآن» و«استيقظت». تحتوي: Start_Time, End_Time, Duration, Serenity_Index, Dream_Signature.

TomorrowNote – رسالة الغد

رسالة مشفّرة طرفيًا تُخزّن مؤقتًا ثم تُرسل لشخص مجهول عند الاستيقاظ، وتحذف فور قراءتها.

3. تدفق المستخدم (User Workflow)

Pre-Sleep

  • postUserIntentionToSleep()
  • selectMoodSymbol(Symbol_ID)
  • joinSleepRoom(Room_ID)
  • storeTomorrowNote(Encrypted_Text)

Post-Sleep

  • endSession()
  • fetchTomorrowNote()
  • calculateSleepMetrics()
  • compareLocalData()

4. الخصوصية والأمان

  • التحليل يتم محليًا قدر الإمكان (On-Device Processing).
  • استخدام Geo_Cell واسعة بدل GPS دقيق لتحقيق k-Anonymity.
  • أدوات واضحة للمستخدم: wipeWeeklyData() واعزل بياناتي.
  • هوية يومية مؤقتة تقلل إمكانية ربط الجلسات ببعضها خارج الجهاز.
